# Что такое Thunk в Redux
> Thunk - это функция, которая задерживает или откладывает выполнение действия. В контексте Redux, `Thunk` используется для обработки **асинхронных** действий внутри действий Redux.

В Redux обычные действия (actions) являются простыми объектами, которые описывают, что должно произойти, но они синхронны по своей природе. Это значит, что Redux не предоставляет нативной поддержки для выполнения асинхронных операций, таких как запросы к серверу или получение данных из базы данных.

Thunk **middleware** позволяет действиям Redux возвращать не только объекты, но и функции. 

Когда функция-действие (так называемый `thunk`) запускается, она получает два параметра: `dispatch` и `getState`. Это позволяет `thunk` 
диспетчеризовать дополнительные действия, и/или получать текущее состояние хранилища Redux.

```jsx
// asyncActionCreator.js
import { fetchData } from './api';

export const fetchUserData = () => {
  return async (dispatch, getState) => {
    try {
      // Выполняем асинхронное действие, например, получение данных с сервера
      const userData = await fetchData();
      
      // Диспетчеризуем обычные синхронные действия с полученными данными
      dispatch({ type: 'FETCH_USER_DATA_SUCCESS', payload: userData });
    } catch (error) {
      dispatch({ type: 'FETCH_USER_DATA_FAILURE', error });
    }
  };
};

```
